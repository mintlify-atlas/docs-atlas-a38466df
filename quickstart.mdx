## Get Started in 5 Minutes

This guide will get you from zero to a fully functional sales intelligence dashboard with realistic data.

<Note>
  **Prerequisites**: Python 3.10-3.12 installed on your system. Check with `python --version`
</Note>

## Installation Steps

<Steps>
  <Step title="Clone the Repository">
    Get the source code from GitHub:

    ```bash
    git clone https://github.com/ricardobing/excel-to-kpi-dashboard.git
    cd excel-to-kpi-dashboard
    ```

    <Note>
      The repository contains all source code, tests, and documentation needed to run the dashboard.
    </Note>
  </Step>

  <Step title="Create Virtual Environment">
    Set up an isolated Python environment:

    <CodeGroup>
      ```bash Windows
      python -m venv .venv
      .venv\Scripts\activate
      ```

      ```bash macOS/Linux
      python -m venv .venv
      source .venv/bin/activate
      ```
    </CodeGroup>

    <Warning>
      Always activate your virtual environment before running commands. Your terminal prompt should show `(.venv)` when active.
    </Warning>
  </Step>

  <Step title="Install Dependencies">
    Install all required Python packages:

    ```bash
    pip install -r visor_kpi/requirements.txt
    ```

    This installs:
    - **streamlit** (1.32+) - Web application framework
    - **pandas** (2.2+) - Data manipulation and analysis
    - **plotly** (5.19+) - Interactive visualizations
    - **openpyxl** (3.1.5+) - Excel file reading
    - **numpy** (1.26+) - Numerical computations
    - **faker** (23.2+) - Mock data generation
    - **pytest** (8.0+) - Testing framework

    <Note>
      Installation typically takes 1-2 minutes depending on your internet connection.
    </Note>
  </Step>

  <Step title="Generate Mock Data">
    Navigate to the application directory and generate sample sales data:

    ```bash
    cd visor_kpi
    python data/mock/generate_mock_data.py
    ```

    This script generates:
    - **12 salespeople** with differentiated performance profiles (star, stable, development, problematic)
    - **180 customers** across 4 commercial channels (Traditional, Supermarkets, Wholesale, HoReCa)
    - **60 products** in 5 categories (Dairy, Grocery, Beverages, Cleaning, Personal Care)
    - **~30,000 transactions** spanning 27 months (Jan 2023 – Mar 2026)
    - **Monthly objectives** per salesperson with 5% annual growth

    The data is saved to `data/raw/mock_data.xlsx` with 5 sheets: Ventas, Vendedores, Clientes, Productos, and Objetivos.

    <Card icon="clock">
      **Generation Time**: 30-60 seconds for complete dataset with validation
    </Card>
  </Step>

  <Step title="Launch the Dashboard">
    Start the Streamlit application:

    ```bash
    streamlit run app.py
    ```

    The dashboard will automatically open in your browser at `http://localhost:8501`

    <Note>
      If the browser doesn't open automatically, copy the URL from the terminal output.
    </Note>
  </Step>
</Steps>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Dashboard shows 'No data available' error">
    Make sure you generated the mock data:

    ```bash
    cd visor_kpi
    python data/mock/generate_mock_data.py
    ```

    Verify the file exists at `visor_kpi/data/raw/mock_data.xlsx`
  </Accordion>

  <Accordion title="ModuleNotFoundError when running the app">
    Ensure your virtual environment is activated and dependencies are installed:

    ```bash
    # Windows
    .venv\Scripts\activate
    # macOS/Linux
    source .venv/bin/activate

    pip install -r visor_kpi/requirements.txt
    ```
  </Accordion>

  <Accordion title="Charts not displaying correctly">
    This is usually a browser compatibility issue. Try:
    - Refreshing the page (Ctrl+F5 or Cmd+Shift+R)
    - Clearing browser cache
    - Using a different browser (Chrome or Firefox recommended)
  </Accordion>

  <Accordion title="Slow performance with filters">
    The first load caches data with Streamlit's `@st.cache_data` decorator. Subsequent filter changes should be instant. If it remains slow:
    - Check your dataset size (should be ~30K rows)
    - Clear Streamlit cache: Press 'C' in the terminal where the app is running
  </Accordion>
</AccordionGroup>
