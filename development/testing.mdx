## Overview

Visor KPI Comercial includes a comprehensive test suite with **51 automated tests** that validate data quality, KPI calculations, Pareto analysis, and alert rules.

<Note>
  All tests are written using **pytest** and cover the core business logic in the `src/` directory, ensuring the application behaves correctly regardless of UI changes.
</Note>

    ```python test_mock_data.py
        ventas = data["ventas"].copy()
        clientes = data["clientes"][["id_cliente", "id_vendedor_asignado"]]
    ```

    ```python test_mock_data.py
    def test_estacionalidad_diciembre_enero(data):
        """December and January have >15% more sales than monthly average."""
        ventas = data["ventas"].copy()
        ventas["mes_num"] = pd.to_datetime(ventas["fecha"]).dt.month
        
        mensual = ventas.groupby(ventas["fecha"].dt.to_period("M"))["importe_neto"].sum()
        promedio = mensual.mean()
        
        meses_dic_ene = ventas[ventas["mes_num"].isin([12, 1])]
        prom_dic_ene = meses_dic_ene.groupby(meses_dic_ene["fecha"].dt.to_period("M"))["importe_neto"].sum().mean()
        
        factor = prom_dic_ene / promedio - 1
        assert factor >= 0.10
    ```

    Validates that the mock data includes realistic seasonal patterns with peak sales in December and January.
  </Accordion>

### 2. KPI Engine Tests (`test_kpis.py`)

Validates all KPI calculation functions in `src/kpi_engine.py` and `src/pareto.py`.

#### Pareto Analysis Tests

```python test_kpis.py
def test_pareto_suma_100():
    """Participation % in Pareto analysis sums to exactly 1.0."""
    from src.pareto import calcular_pareto
    df = make_simple_df()
    grp = df.groupby("id_producto")["importe_neto"].sum().reset_index()
    result = calcular_pareto(grp, "importe_neto", "id_producto")
    total_pct = result["participacion_pct"].sum()
    assert abs(total_pct - 1.0) < 0.01

def test_pareto_categoria_A_cubre_80pct():
    """Category A covers 70-90% of total volume (80/20 rule)."""
    result = calcular_pareto(grp, "importe_neto", "id_producto")
    vol_a = result[result["categoria_pareto"] == "A"]["importe_neto"].sum()
    pct_a = vol_a / result["importe_neto"].sum()
    assert 0.70 <= pct_a <= 0.90

def test_pareto_acumulado_monotono():
    """Cumulative % must be monotonically increasing."""
    diffs = result["acumulado_pct"].diff().dropna()
    assert (diffs >= -0.001).all()
```

## Continuous Integration

### GitHub Actions Workflow

Example CI configuration for automated testing:

```yaml .github/workflows/test.yml
name: Test Suite

on: [push, pull_request]

jobs:
  test:
    runs-on: ubuntu-latest
    strategy:
      matrix:
        python-version: ['3.10', '3.11', '3.12']
    
    steps:
      - uses: actions/checkout@v3
      
      - name: Set up Python ${{ matrix.python-version }}
        uses: actions/setup-python@v4
        with:
          python-version: ${{ matrix.python-version }}
      
      - name: Install dependencies
        run: |
          cd visor_kpi
          pip install -r requirements.txt
      
      - name: Generate mock data
        run: |
          cd visor_kpi
          python data/mock/generate_mock_data.py
      
      - name: Run tests
        run: |
          cd visor_kpi
          pytest tests/ -v --tb=short
```

## Writing New Tests

### Test File Naming

- Test files must start with `test_` (e.g., `test_new_feature.py`)
- Test functions must start with `test_` (e.g., `def test_calculation():`)

### Example Test

```python tests/test_new_feature.py
import pytest
import pandas as pd
from src.kpi_engine import nueva_funcion

def test_nueva_funcion_retorna_dataframe():
    """nueva_funcion should return a DataFrame."""
    df = pd.DataFrame({"col": [1, 2, 3]})
    result = nueva_funcion(df)
    assert isinstance(result, pd.DataFrame)

def test_nueva_funcion_maneja_datos_vacios():
    """nueva_funcion should handle empty DataFrames gracefully."""
    df_empty = pd.DataFrame()
    result = nueva_funcion(df_empty)
    assert len(result) == 0
```
