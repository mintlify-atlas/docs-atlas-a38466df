## System Requirements

Before installing Visor KPI Comercial, ensure your system meets these requirements:

    - Windows 10/11
    - macOS 11+ (Big Sur or later)

## Verify Python Installation

Check your Python version:

<CodeGroup>
  ```bash Windows
  python --version
  # Should output: Python 3.10.x, 3.11.x, or 3.12.x
  ```

  ```bash macOS/Linux
  python3 --version
  # Should output: Python 3.10.x, 3.11.x, or 3.12.x
  ```
</CodeGroup>

<Warning>
  If Python is not installed or the version is incompatible:
  - **Windows/macOS**: Download from [python.org](https://www.python.org/downloads/)
  - **Linux**: Use your package manager (e.g., `apt install python3.11`)
</Warning>

## Step-by-Step Installation

<Steps>
  <Step title="Clone the Repository">
    Download the source code from GitHub:

    ```bash
    git clone https://github.com/ricardobing/excel-to-kpi-dashboard.git
    cd excel-to-kpi-dashboard
    ```

    <Accordion title="Don't have Git?">
      You can download the repository as a ZIP file:
      1. Visit [https://github.com/ricardobing/excel-to-kpi-dashboard](https://github.com/ricardobing/excel-to-kpi-dashboard)
      2. Click the green "Code" button
      3. Select "Download ZIP"
      4. Extract the ZIP file to your desired location
      5. Open a terminal in the extracted directory
    </Accordion>
  </Step>

  <Step title="Create Virtual Environment">
    A virtual environment isolates project dependencies from your system Python:

    <CodeGroup>
      ```bash Windows (Command Prompt)
      python -m venv .venv
      .venv\Scripts\activate
      ```

      ```bash Windows (PowerShell)
      python -m venv .venv
      .venv\Scripts\Activate.ps1
      ```

      ```bash macOS/Linux
      python3 -m venv .venv
      source .venv/bin/activate
      ```
    </CodeGroup>

    When activated, your terminal prompt should show `(.venv)` at the beginning.

    <Note>
      **PowerShell Execution Policy Error?**
      
      If you see "cannot be loaded because running scripts is disabled":
      ```powershell
      Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
      ```
    </Note>
  </Step>

  <Step title="Install Dependencies">
    Navigate to the application directory and install all required packages:

    ```bash
    cd visor_kpi
    pip install -r requirements.txt
    ```

      ```python
      PERFILES_FACTOR = {

          "estable":      {"base": 0.95, "sd": 0.08, "mejora": 0.001},
          "desarrollo":   {"base": 0.75, "sd": 0.10, "mejora": 0.005},
          "problematico": {"base": 0.62, "sd": 0.15, "mejora": 0.000},
      }

      # Growth trend: 8% annual ≈ 0.64% monthly compound
      def _factor_crecimiento(mes: date) -> float:
          meses_transcurridos = (
              (mes.year - FECHA_INICIO.year) * 12 + 
              (mes.month - FECHA_INICIO.month)
      ```

## Formatting Functions

The system includes utility functions for consistent number formatting:

```python config.py
def fmt_moneda(valor: float) -> str:
    """Format as currency: $1.284.500 (Argentine style)"""
    if valor >= 1_000:
        s = f"{abs(valor):,.0f}".replace(",", ".")
    else:
        s = f"{abs(valor):.2f}".replace(".", ",")
    return f"{'-' if valor < 0 else ''}${s}"

def fmt_pct(valor: float, decimales: int = 1) -> str:
    """Format as percentage: 84.2%"""
    return f"{float(valor) * 100:.{decimales}f}%"

def fmt_numero(valor: float) -> str:
    """Format with thousands separator: 1.284"""
    return f"{float(valor):,.0f}".replace(",", ".")
```

## Data Loading and Caching

The ETL pipeline is defined in `src/data_loader.py`:

```python src/data_loader.py
import streamlit as st
import pandas as pd
from config import DATA_PATH

@st.cache_data(ttl=3600, show_spinner=False)
def load_data() -> dict[str, pd.DataFrame]:
    """
    Reads Excel and returns dict with parsed DataFrames.
    Keys: ventas, vendedores, clientes, productos, objetivos
    
    Cached for 1 hour (3600 seconds) to improve performance.
    """
    if not os.path.exists(DATA_PATH):
        raise FileNotFoundError(
            f"No se encontró el archivo de datos: {DATA_PATH}\n"
            "Ejecutá primero: python data/mock/generate_mock_data.py"
        )

    xl = pd.ExcelFile(DATA_PATH)

    # Parse Ventas sheet
    ventas = xl.parse("Ventas")
    ventas["fecha"] = pd.to_datetime(ventas["fecha"])
    ventas["año"]   = ventas["fecha"].dt.year
    ventas["mes"]   = ventas["fecha"].dt.month
    ventas["periodo"] = ventas["fecha"].dt.to_period("M")
    
    # Ensure numeric columns
    for col in ["importe_neto", "importe_bruto", "cantidad", 
                "precio_unitario", "descuento_pct"]:
        if col in ventas.columns:
            ventas[col] = pd.to_numeric(ventas[col], errors="coerce").fillna(0)

    # Parse other sheets...
    vendedores = xl.parse("Vendedores")
    clientes   = xl.parse("Clientes")
    productos  = xl.parse("Productos")
    objetivos  = xl.parse("Objetivos")

    return {
        "ventas":     ventas,
        "vendedores": vendedores,
        "clientes":   clientes,
        "productos":  productos,
        "objetivos":  objetivos,
    }
```

<Note>
  The `@st.cache_data` decorator caches the loaded data for 1 hour, significantly improving performance on subsequent page loads and filter changes.
</Note>

## KPI Engine

Core business logic is in `src/kpi_engine.py`:

```python src/kpi_engine.py
import pandas as pd
from src.pareto import calcular_pareto

def kpi_ventas_periodo(
    df: pd.DataFrame,
    df_anterior: pd.DataFrame | None = None,
    objetivos_df: pd.DataFrame | None = None,
    fecha_desde: date | None = None,
    fecha_hasta: date | None = None,
) -> dict:
    """
    Aggregate sales KPIs for the period.

    Returns dict with:
        importe_total, importe_vs_anterior, objetivo_total,
        cumplimiento_pct, transacciones_count, ticket_promedio
    """
    importe_total = float(df["importe_neto"].sum()) if not df.empty else 0.0
    n_transacciones = len(df)
    clientes_unicos = df["id_cliente"].nunique() if not df.empty else 0
    ticket_prom = (importe_total / clientes_unicos) if clientes_unicos > 0 else 0.0

    # Calculate period objective
    objetivo_total = 0.0
    if objetivos_df is not None and not objetivos_df.empty:
        mask = (
            (objetivos_df["periodo"] >= pd.Timestamp(fecha_desde)) &
            (objetivos_df["periodo"] <= pd.Timestamp(fecha_hasta))
        )
        objetivo_total = float(objetivos_df[mask]["objetivo"].sum())

    cumplimiento = importe_total / objetivo_total if objetivo_total > 0 else None

    # Delta vs previous period
    importe_vs_ant = None
    if df_anterior is not None and not df_anterior.empty:
        imp_ant = float(df_anterior["importe_neto"].sum())
        if imp_ant > 0:
            importe_vs_ant = (importe_total - imp_ant) / imp_ant

    return {
        "importe_total":       importe_total,
        "importe_vs_anterior": importe_vs_ant,
        "objetivo_total":      objetivo_total,
        "cumplimiento_pct":    cumplimiento,
        "transacciones_count": n_transacciones,
        "ticket_promedio":     ticket_prom,
        "clientes_unicos":     clientes_unicos,
    }
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Import errors when running app.py">
    Ensure you're running from the correct directory:

    ```bash
    # You should be in the visor_kpi directory
    pwd  # or cd on Windows
    # Output should end with: /visor_kpi

    # Run from here
    streamlit run app.py
    ```

    If imports still fail, check your PYTHONPATH:
    ```python
    import sys
    print(sys.path)
    ```
  </Accordion>

  <Accordion title="'data/raw/mock_data.xlsx' not found">
    The mock data file must exist before running the app:

    ```bash
    # Generate it first
    python data/mock/generate_mock_data.py

    # Verify it was created
    ls data/raw/
    # Should show: mock_data.xlsx
    ```
  </Accordion>

  <Accordion title="Port 8501 already in use">
    Another Streamlit app is running. Either:

    1. **Stop the other app**: Press Ctrl+C in its terminal
    2. **Use a different port**:
       ```bash
       streamlit run app.py --server.port 8502
       ```
  </Accordion>

  <Accordion title="Performance issues with large datasets">
    If using your own data with >100K rows:

    1. **Increase cache TTL**:
       ```python
       @st.cache_data(ttl=7200)  # 2 hours
       def load_data():
           ...
       ```

    2. **Add date filters earlier in the pipeline**:
       ```python
       # Filter before joins to reduce dataframe size
       ventas = ventas[
           (ventas["fecha"] >= pd.Timestamp("2025-01-01"))
       ]
       ```

    3. **Consider using Parquet instead of Excel**:
       ```python
       df.to_parquet("data.parquet", compression="gzip")
       ```
  </Accordion>

  <Accordion title="Excel file locked on Windows">
    If you see "Permission denied" when regenerating data:

    1. Close Excel if `mock_data.xlsx` is open
    2. Delete the existing file:
       ```bash
       del data\raw\mock_data.xlsx
       ```
    3. Regenerate:
       ```bash
       python data/mock/generate_mock_data.py
       ```
  </Accordion>
</AccordionGroup>
