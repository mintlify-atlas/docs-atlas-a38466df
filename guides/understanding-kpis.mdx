## Sales KPIs

### Function: `kpi_ventas_periodo()`

**Source**: `kpi_engine.py:20-65`

<AccordionGroup>
  <Accordion title="Importe Total (Total Sales)" icon="dollar-sign">
    **Formula**:
    ```python
importe_total = df["importe_neto"].sum()
    ```
    
    **Description**: Sum of all net sales amounts for the filtered period
    
    **Data Type**: Float (currency)
    
    **Example**: $1,234,567.89
  </Accordion>

  <Accordion title="Importe vs. Anterior (YoY/MoM Change)" icon="chart-line">
    **Formula**:
    ```python
importe_vs_anterior = (importe_total - importe_anterior) / importe_anterior
    ```
    
    **Description**: Percentage change compared to previous equivalent period
    
    **Data Type**: Float (percentage, can be negative)
    
    **Display**: +15.3% or -8.2%
    
    **Note**: Returns `None` if no previous period data available
  </Accordion>

  <Accordion title="Objetivo Total (Total Target)" icon="bullseye">
    **Formula**:
    ```python
objetivo_total = objetivos_df[
    (periodo >= fecha_desde) & (periodo <= fecha_hasta)
]["objetivo"].sum()
    ```
    
    **Description**: Sum of all salesperson targets for the period
    
    **Source**: `Objetivos` sheet
    
    **Data Type**: Float (currency)
  </Accordion>

  <Accordion title="Cumplimiento (Target Achievement)" icon="percent">
    **Formula**:
    ```python
cumplimiento_pct = importe_total / objetivo_total
    ```
    
    **Description**: Percentage of target achieved
    
    **Range**: 0.0 to ∞ (typically 0.0 to 1.5)
    
    **Display**: 94.3%
    
    **Traffic Light**:
    - 🟢 Green: ≥100% (on or above target)
    - 🟡 Orange: 80-99% (close to target)
    - 🔴 Red: &lt;80% (below target)
    
    **Code** (`kpi_engine.py:431-448`):
    ```python
    def _label_semaforo(cumpl: float | None) -> str:
        if cumpl is None:
            return "danger"
        if cumpl >= 1.00:
            return "success"
        elif cumpl >= 0.80:
            return "warning"
        return "danger"
    ```
  </Accordion>

  <Accordion title="Transacciones Count (Transaction Count)" icon="receipt">
    **Formula**:
    ```python
transacciones_count = len(df)
    ```
    
    **Description**: Total number of sales transactions
    
    **Data Type**: Integer
    
    **Use Case**: Activity level indicator, frequency analysis
  </Accordion>

  <Accordion title="Ticket Promedio (Average Ticket)" icon="ticket">
    **Formula**:
    ```python
ticket_promedio = importe_total / clientes_unicos
    ```
    
    **Description**: Average sales amount **per unique customer** (not per transaction)
    
    **Data Type**: Float (currency)
    
    **Interpretation**: Higher ticket = larger order sizes or higher-value customers
    
    <Warning>
    This is **per customer**, not per transaction. For transaction-level average, use: `importe_total / transacciones_count`
    </Warning>
  </Accordion>

  <Accordion title="Clientes Únicos (Unique Customers)" icon="user-group">
    **Formula**:
    ```python
clientes_unicos = df["id_cliente"].nunique()
    ```
    
    **Description**: Count of distinct customers who made purchases
    
    **Data Type**: Integer
    
    **Use Case**: Customer reach, market penetration
  </Accordion>
</AccordionGroup>

## Margin KPIs

### Function: `kpi_margen()`

**Source**: `kpi_engine.py:68-92`

<AccordionGroup>
  <Accordion title="Margen Bruto Total (Total Gross Margin)" icon="sack-dollar">
    **Formula**:
    ```python
margen_neto = importe_neto - (costo_catalogo / precio_catalogo * importe_neto)
margen_bruto_total = df["margen_neto"].sum()
    ```
    
    **Description**: Total profit after subtracting product costs
    
    **Calculation Details**:
    1. Margin per transaction calculated in master dataframe (`data_loader.py:121`)
    2. Cost ratio applied: `costo/precio`
    3. Applied to net amount: `margen_neto = importe_neto × (1 - costo_ratio)`
    
    **Data Type**: Float (currency)
  </Accordion>

  <Accordion title="Margen % Promedio (Average Margin %)" icon="percent">
    **Formula**:
    ```python
margen_pct_promedio = margen_bruto_total / importe_total
    ```
    
    **Description**: Overall margin percentage across all sales
    
    **Range**: 0.0 to 1.0 (typically 0.15 to 0.45)
    
    **Display**: 32.5%
    
    **Interpretation**:
    - &lt;20%: Low margin (volume business)
    - 20-40%: Healthy margin
    - >40%: High margin (premium products)
  </Accordion>

  <Accordion title="Margen vs. Anterior (Margin Change)" icon="arrow-trend-up">
    **Formula**:
    ```python
margen_vs_anterior = (margen_total - margen_anterior) / margen_anterior
    ```
    
    **Description**: Percentage change in total margin vs. previous period
    
    **Data Type**: Float (percentage)
    
    **Insights**:
    - Positive: Improved profitability
    - Negative: Margin pressure (discounting, cost increases, product mix shift)
  </Accordion>
</AccordionGroup>

## Coverage KPIs

### Function: `kpi_cobertura_clientes()`

**Source**: `kpi_engine.py:95-135`

<AccordionGroup>
  <Accordion title="Clientes Activos Período (Active Customers in Period)" icon="user-check">
    **Formula**:
    ```python
compraron = set(df["id_cliente"].unique())
clientes_activos_total = clientes_df[clientes_df["activo"] == True]["id_cliente"]
n_activos_periodo = len(compraron & set(clientes_activos_total))
    ```
    
    **Description**: Number of active customers who made at least one purchase in the period
    
    **Data Type**: Integer
  </Accordion>

  <Accordion title="Clientes Total (Total Customer Base)" icon="users">
    **Formula**:
    ```python
clientes_total = len(clientes_df[clientes_df["activo"] == True])
    ```
    
    **Description**: Total count of active customers in database
    
    **Source**: `Clientes` sheet where `activo = TRUE`
  </Accordion>

  <Accordion title="Cobertura % (Coverage Percentage)" icon="chart-pie">
    **Formula**:
    ```python
cobertura_pct = clientes_activos_periodo / clientes_total
    ```
    
    **Description**: Percentage of active customer base that purchased in period
    
    **Range**: 0.0 to 1.0
    
    **Display**: 67.8%
    
    **Benchmarks**:
    - &lt;50%: Low engagement, opportunity to activate dormant customers
    - 50-70%: Good coverage
    - >70%: Excellent engagement
  </Accordion>

  <Accordion title="Clientes Perdidos (Lost Customers)" icon="user-xmark">
    **Formula**:
    ```python
compraron_anterior = set(df_anterior["id_cliente"].unique())
clientes_perdidos = len(compraron_anterior - compraron)
    ```
    
    **Description**: Customers who bought in previous period but NOT in current period
    
    **Data Type**: Integer
    
    **Alert Trigger**: High churn indicator (see Alert ALT002)
  </Accordion>

  <Accordion title="Clientes Nuevos (New Customers)" icon="user-plus">
    **Formula**:
    ```python
todos_anteriores = set(df_anterior["id_cliente"].unique())
clientes_nuevos = len(compraron - todos_anteriores)
    ```
    
    **Description**: Customers who made their FIRST purchase in current period
    
    **Data Type**: Integer
    
    **Use Case**: Customer acquisition rate, market expansion
  </Accordion>

  <Accordion title="Clientes Recuperados (Recovered Customers)" icon="user-check">
    **Formula**:
    ```python
clientes_recuperados = len(
    compraron & compraron_anterior - 
    (compraron_anterior & set(clientes_activos_total))
)
    ```
    
    **Description**: Previously inactive customers who made a purchase in current period
    
    **Data Type**: Integer
    
    **Use Case**: Re-engagement success metric
  </Accordion>
</AccordionGroup>

    # Sort by revenue descending
    df_sorted = df.sort_values(metric, ascending=False)
    # Calculate cumulative percentage
    df_sorted["pct_acumulado"] = df_sorted[metric].cumsum() / total
    # Assign categories
    categoria_pareto = [

## Performance Considerations

<Note>
**Caching Strategy**

All KPI functions receive filtered DataFrames as input. The filtering itself is cached via `@st.cache_data` on `get_filtered_data()`.

KPI calculations are NOT cached separately - they're fast enough to recalculate on demand.
</Note>

### Optimization Tips

1. **Vectorized Operations**: All KPIs use pandas vectorized operations (`.sum()`, `.groupby()`, `.nunique()`) for speed

2. **Single Pass**: Multiple KPIs are calculated in one groupby operation where possible

3. **Lazy Evaluation**: KPIs are only calculated for pages that need them

## Technical Reference

### Return Types

| Function | Returns | Use Case |
|----------|---------|----------|
| `kpi_ventas_periodo()` | `dict` | Summary metrics |
| `kpi_margen()` | `dict` | Summary metrics |
| `kpi_cobertura_clientes()` | `dict` | Summary metrics |
| `kpi_por_vendedor()` | `DataFrame` | Ranking tables |

| `kpi_clientes()` | `DataFrame` | Customer analysis |
| `kpi_evolucion_mensual()` | `DataFrame` | Time series charts |

### Common Parameters

- `df: pd.DataFrame` - Filtered sales data (current period)
- `df_anterior: pd.DataFrame | None` - Previous period data for comparisons
- `objetivos_df: pd.DataFrame | None` - Targets from `Objetivos` sheet
- `clientes_df: pd.DataFrame` - Customer master from `Clientes` sheet
- `productos_df: pd.DataFrame` - Product master from `Productos` sheet
- `fecha_desde: date | None` - Period start date
- `fecha_hasta: date | None` - Period end date
